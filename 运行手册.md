# 中国古建筑类国保时空分布平台
## 运行手册

文档版本：V1.0  
编制日期：2026-04-08

---

## 1. 运行环境要求

### 1.1 基础环境
- Node.js >= 18
- npm >= 9
- 操作系统：Windows / Linux / macOS

### 1.2 目录要求
项目根目录应包含：
- `package.json`
- `src/`
- `public/`
- `vite.config.ts`

---

## 2. 首次运行步骤

### 2.1 安装依赖
在项目根目录执行：
```bash
npm install
```

### 2.2 启动开发服务
```bash
npm run dev
```

### 2.3 访问地址
默认访问：
- `http://127.0.0.1:8112/#/index`

---

## 3. 构建与预览

### 3.1 生产构建
```bash
npm run build
```

### 3.2 本地预览
```bash
npm run preview
```

### 3.3 类型检查
```bash
npm run type-check
```

---

## 4. 数据脚本操作

### 4.1 同步公开数据
```bash
npm run sync:relics
```
执行后会更新 `public/data/` 下数据文件。

### 4.2 CSV 导入
```bash
npm run import:2160 -- --input ./your.csv --output ./public/data/gujian_2160.json
```

---

## 5. Mock 与真实接口切换

### 5.1 默认行为
系统默认启用 mock（`src/main.ts`）：
```ts
import { mockXHR } from "@/mock/index";
mockXHR();
```

### 5.2 切换真实接口
1. 注释 `mockXHR()` 相关代码
2. 在 `src/api/` 中配置真实接口地址与参数
3. 重启开发服务并联调验证

---

## 6. 常见故障处理

### 6.1 启动失败：依赖缺失
处理：
```bash
rm -rf node_modules
npm install
```

### 6.2 页面空白或路由异常
处理：
- 检查 Node 版本
- 执行 `npm run type-check`
- 确认 `src/router/index.ts` 未被误改

### 6.3 地图不显示
处理：
- 检查 `public/map-geojson/china.json` 是否存在
- 检查浏览器控制台报错

### 6.4 检索无数据
处理：
- 检查 `public/data/gujian_5061_latest.json`
- 校验 JSON 文件格式是否正确

### 6.5 3D 页面加载慢
处理：
- 模型文件体积较大，属正常现象
- 比赛现场可提前打开页面预热缓存

---

## 7. 部署操作指引

### 7.1 构建
```bash
npm run build
```

### 7.2 部署
将 `dist/` 目录发布到静态服务器。

### 7.3 子路径部署
如部署到子目录，需按实际路径调整 `vite.config.ts` 中 `base` 配置。
