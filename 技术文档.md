## 1. 项目概述

中国古建筑类国保时空分布平台是一个基于 Vue 3 + Vite 的前端可视化系统，围绕古建筑类全国重点文物保护单位数据，构建集“展示、检索、联动分析、决策推演”于一体的业务展示平台。

系统主要面向以下场景：
- 古建筑数据可视化展示与讲解
- 省级/地市级空间分布分析
- 保护优先级模型演示
- 比赛答辩与成果路演

---

## 2. 技术架构

### 2.1 总体架构
系统采用前端单页应用（SPA）架构，静态资源部署模式：
- 表现层：Vue 组件 + Element Plus
- 状态层：Pinia
- 路由层：Vue Router（Hash 路由）
- 图表层：ECharts / vue-echarts
- 数据层：本地 JSON 数据文件 + 可选 Mock 接口
- 脚本层：Node.js 数据处理脚本

### 2.2 运行形态
- 开发运行：Vite Dev Server
- 生产部署：Vite 构建为静态文件后部署到 Nginx/静态服务器

## 4. 功能模块技术实现

### 4.1 首页大屏（`/#/index`）
核心实现文件：
- `src/views/index/index.vue`
- `src/views/index/center-map.vue`
- `src/views/index/right-top.vue`
- `src/views/index/compare-link.ts`

技术要点：
- 三栏组合布局
- 地图、排行、趋势多模块联动
- 事件总线 + 共享状态联合实现联动
- 口径追溯抽屉组件复用

### 4.2 全局检索（`/#/search`）
核心实现文件：
- `src/views/search/index.vue`

技术要点：
- 基于本地数据集的多条件筛选
- 分页与数据回退机制
- 支持检索结果联动回首页地图/排行

### 4.3 决策沙盘（`/#/decision-sandbox`）
核心实现文件：
- `src/views/decision-sandbox/index.vue`

技术要点：
- 预算/队伍/工期等约束参数建模
- 方案 A/B 求解与指标输出
- 结果导出（CSV/TXT）

### 4.4 口径追溯模块
核心实现文件：
- `src/components/data-lineage-panel/data-lineage-panel.vue`

技术要点：
- 为面板统一提供来源、时间、公式、样本量、局限性信息
- 提升结果可追溯性与答辩可解释性

### 4.5 独立展示页
- `public/source-catalog.html`：来源目录说明
- `public/3d-showroom.html`：Three.js 3D 展厅
- `public/video-viewer.html`：视频播放页

### 5.1 数据目录
- `public/data/`：业务数据
- `public/map-geojson/`：地图边界数据

### 5.2 关键数据文件
- `gujian_5061_latest.json`：检索页默认主数据
- `gujian_2170_latest.json`：古建筑子集
- `gujian_5058.json` / `gujian_2160.json`：兼容口径

- 脚本：`scripts/fetch_relics_from_arcgis.mjs`
- 作用：从 ArcGIS 公开服务拉取并生成标准化数据文件

## 8. 关键技术说明

### 8.1 Mock 机制
系统默认在 `src/main.ts` 启用 mock：
```ts
import { mockXHR } from "@/mock/index";
mockXHR();
```
如需接入真实后端，可注释上述两行并在 `src/api/` 配置真实接口。

### 8.2 路由模式
系统采用 Hash 路由，兼容静态部署路径。

### 8.3 可追溯性设计
通过来源抽屉组件和来源目录页，实现“结论-来源-口径”闭环说明。

## 9. 风险与改进建议

### 9.1 当前风险
- 前端直连问答服务存在密钥暴露风险
- `public/models/` 资源体积较大，影响首次加载
- 事件联动依赖窗口事件，后续可收敛为统一状态流

### 9.2 建议优化方向
- 接口代理后端化，敏感信息迁移至服务端
- 资源按需加载与模型分片加载
- 增加端到端测试与部署前检查流程
